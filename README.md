## ⚡ Quick Start

### Prerequisites

```bash
# 1. Install Ollama → https://ollama.ai
# 2. Pull the Gemma 4 model
ollama pull gemma4

# 3. Clone this repo
git clone https://github.com/kennedyraju55/90-local-llm-projects.git
cd 90-local-llm-projects

# 4. Install base dependencies
pip install -r requirements-base.txt
```

### Run Any Project

```bash
# Pick any project (1-100) and run it!
cd 01-pdf-chat-assistant
pip install -r requirements.txt
python app.py --help
```

> 💡 **Each of the 100 projects also lives in its own dedicated GitHub repo** — click any project link below to explore it individually!

<summary>💡 <strong>Why Attorneys Need Local LLMs</strong></summary>

<br/>

**The Problem:** Cloud-based AI tools pose a direct threat to attorney-client privilege. When you paste a client's contract into ChatGPT or a cloud API, that data is transmitted to — and potentially stored on — third-party servers. This can constitute a waiver of privilege.

**The Solution:** Every legal tool in this repo runs **100% locally** via Ollama + Gemma 4. Your client's sensitive documents, contracts, case files, and legal strategies **never leave your machine**. Period.

**Use Cases:**
- 📋 **Contract Review:** Analyze NDAs, employment agreements, and vendor contracts without exposing terms
- 🔍 **Case Research:** Search through case law and identify relevant precedents locally
- 📝 **Brief Drafting:** Generate legal briefs with proper citation formatting
- 🎤 **Deposition Analysis:** Summarize hours of testimony into key findings
- ✅ **Compliance Audits:** Check policies against GDPR, CCPA, and other regulations

<details>
<summary>💡 <strong>Why Healthcare Professionals Need Local LLMs</strong></summary>

<br/>

**The Problem:** HIPAA regulations strictly govern how Protected Health Information (PHI) is transmitted and stored. Cloud-based AI services require Business Associate Agreements (BAAs), carry breach notification risks, and create audit complexity. Many smaller practices simply cannot navigate this.

**The Solution:** Every healthcare tool here processes data **entirely on your local machine**. Patient names, diagnoses, medications, lab results — nothing is ever transmitted. This means:

- ✅ **No BAA required** — data never reaches a third party
- ✅ **No breach risk** — nothing to breach if nothing is transmitted
- ✅ **Works offline** — ideal for rural clinics or air-gapped environments
- ✅ **Audit-friendly** — complete data locality simplifies compliance

**Use Cases:**
- 🩺 **Clinical Documentation:** Generate SOAP notes, discharge summaries, and referral letters
- 🧬 **Differential Diagnosis:** Get ranked DDx suggestions from symptom presentations
- 💊 **Drug Safety:** Check multi-drug interactions before prescribing
- 🔬 **Lab Interpretation:** Explain complex lab panels to patients in plain language
- 📋 **Intake Processing:** Summarize lengthy patient intake forms into concise clinical briefs
- 🔒 **De-identification:** Strip PHI from records before sharing for research

> ⚠️ **Disclaimer:** These tools are for clinical decision **support** only. They do not replace professional medical judgment. Always verify outputs against established clinical guidelines.

</details>
